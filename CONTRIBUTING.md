# Contributing to PIIIX

Thanks for your interest in contributing! This document covers how to make a
contribution, and how to apply for an ongoing role on a PIIIX project.

## Making a contribution

1. Fork the repository (`chapters` or `vectory`) and open a pull request.
2. On your first pull request, a bot will comment asking you to sign the
   [Individual CLA](https://github.com/PIIIX-org/.github/blob/main/CLA.md).
   This is required before any pull request can be merged — to sign:
   1. Read the CLA at the link above.
   2. Reply to the bot's comment on your pull request with a comment
      containing this sentence: `I have read the CLA Document and I hereby
      sign the CLA.`
   3. On the same comment, add your full legal name and an email address
      where you can be reached, for example:

      ```
      I have read the CLA Document and I hereby sign the CLA.
      Full legal name: Jane Doe
      Email: jane@example.com
      ```

   4. The bot updates your pull request's status automatically within about
      a minute — no need to comment "recheck" unless it hasn't updated after
      a few minutes.
3. A Reviewer or Manager will review your pull request. Please be responsive
   to feedback.

## Roles

PIIIX-org uses the following roles, each backed by a GitHub team:

| Role | Access | What they do |
|---|---|---|
| Manager | Maintain | Repo settings, branch protection, merges |
| Developer | Write | Pushes code, opens/merges pull requests |
| Reviewer | Triage | Reviews pull requests, triages issues |
| Designer | Write | Pushes UI/asset/CSS pull requests |
| Docs/Content | Write | Pushes docs/README changes |
| Community/Support | Triage | Labels and triages issues, helps other contributors |

Everyone starts as an outside contributor via fork-and-PR. No one is added to
a team without meeting the criteria below — access isn't handed out at
random.

## Becoming a Developer, Designer, Docs/Content, Reviewer, or Community/Support contributor

You're eligible to apply once you have:

- **Developer / Designer / Docs-Content**: at least 2 merged pull requests to
  a PIIIX-org public repository, following this document's guidelines.
- **Reviewer**: at least 2 merged pull requests, or a track record of
  substantive review comments on others' pull requests.
- **Community/Support**: a track record of helpful activity in Issues or
  Discussions — no pull request requirement.

To apply, open a **Contributor Application** issue (use the issue template)
linking your relevant pull requests or activity and the role you're applying
for. A Manager will review it, typically within a week.

The Manager role is not self-applied for — it's extended directly by the
maintainer based on sustained trust.

## Code of conduct

Be respectful, be constructive, assume good faith. Disagreements about code
are fine; personal attacks are not.
